Thoughts

When deciding on how my workflow for adding posts and deployment I wanted a few core things.

  • write posts in markdown
  • statically render to html
  • easy to deploy with hands off approach for the most part

Since I’ve been writing a lot more Go recently, i found Hugo a good choice. It covered most of my requirements out of the box. The only hugo doesnt take care of out of the box is easy mostly hands off deploy.

For deployment I chose github actions and a handy github action provider peaceiris/actions-gh-pages

Whats next

I’ve been working on a prototype for scripting part of Golang from Javascript using v8go. Not sure what I intend to get out of this yet or the bigger project but it sounds fun and challenging.
